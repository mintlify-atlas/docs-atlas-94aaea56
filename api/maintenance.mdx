## Overview

The Maintenance API provides endpoints for managing preventive and corrective maintenance operations, work orders, routines, schedules, and maintenance notifications.

**Base Path:** `/mantenimiento/`

## Work Order Search

### Search Work Orders

Search for work orders by various criteria.

```http
GET /mantenimiento/api/search-ordenes/
```

<ParamField query="q" type="string">
  Search query for work order details
</ParamField>

<ParamField query="estado" type="string">
  Filter by status: PENDIENTE, EN_PROGRESO, COMPLETADA, CANCELADA
</ParamField>

<ParamField query="tipo" type="string">
  Filter by type: PREVENTIVA, CORRECTIVA
</ParamField>

<ParamField query="fecha_desde" type="string">
  Start date filter (YYYY-MM-DD)
</ParamField>

<ParamField query="fecha_hasta" type="string">
  End date filter (YYYY-MM-DD)
</ParamField>

#### Response

<ResponseField name="results" type="array">
  Array of work order objects
  <Expandable title="Work Order Object">
    <ResponseField name="id" type="integer">
      Work order ID
    </ResponseField>
    <ResponseField name="codigo" type="string">
      Work order code
    </ResponseField>
    <ResponseField name="tipo" type="string">
      Type: PREVENTIVA or CORRECTIVA
    </ResponseField>
    <ResponseField name="estado" type="string">
      Current status
    </ResponseField>
    <ResponseField name="rutina_nombre" type="string">
      Associated routine name (for preventive)
    </ResponseField>
    <ResponseField name="inicio_programado" type="string">
      Scheduled start date/time
    </ResponseField>
    <ResponseField name="activos" type="array">
      Array of associated asset IDs
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>

```javascript Request
fetch('/mantenimiento/api/search-ordenes/?estado=PENDIENTE&tipo=PREVENTIVA', {
  credentials: 'same-origin'
})
.then(res => res.json())
.then(data => console.log(data.results));
```

```json Response
{
  "results": [
    {
      "id": 1523,
      "codigo": "OT-2024-1523",
      "tipo": "PREVENTIVA",
      "estado": "PENDIENTE",
      "rutina_nombre": "Monthly Pump Inspection",
      "inicio_programado": "2024-03-15T08:00:00Z",
      "activos": [123, 124, 125]
    }
  ]
}
```

</CodeGroup>

## Update Work Order

### Update Work Order Status and Notes

Update the status and notes of a work order.

```http
POST /mantenimiento/api/ot/{pk}/update/
```

<ParamField path="pk" type="integer" required>
  Work order ID
</ParamField>

<ParamField body="estado" type="string">
  New status: PENDIENTE, EN_PROGRESO, COMPLETADA, CANCELADA
</ParamField>

<ParamField body="notas" type="string">
  Updated notes
</ParamField>

<ParamField body="inicio_real" type="string">
  Actual start datetime (ISO format)
</ParamField>

<ParamField body="fin_real" type="string">
  Actual end datetime (ISO format)
</ParamField>

#### Response

<ResponseField name="status" type="string">
  "success" or "error"
</ResponseField>

<ResponseField name="message" type="string">
  Status message
</ResponseField>

<CodeGroup>

```javascript Request
fetch('/mantenimiento/api/ot/1523/update/', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-CSRFToken': csrfToken
  },
  credentials: 'same-origin',
  body: JSON.stringify({
    estado: 'COMPLETADA',
    fin_real: new Date().toISOString(),
    notas: 'Inspection completed. All systems normal.'
  })
});
```

```json Response
{
  "status": "success",
  "message": "Work order updated successfully"
}
```

</CodeGroup>

### Update Work Order Date

```http
POST /mantenimiento/api/update-ot-date/
```

<ParamField body="ot_id" type="integer" required>
  Work order ID

<ParamField body="new_date" type="string" required>

</ParamField>

<ParamField body="reason" type="string">

### Bulk Update Work Order Dates

```http

```

<ParamField body="updates" type="array" required>

    <ParamField body="ot_id" type="integer" required>

    </ParamField>
    <ParamField body="new_date" type="string" required>
      New scheduled date
    </ParamField>
  </Expandable>
</ParamField>


<ParamField body="ot_id" type="integer" required>

</ParamField>

### Delete Work Orders

Delete multiple work orders.

```http
POST /mantenimiento/api/delete-ots/
```

<ParamField body="ot_ids" type="array" required>

</ParamField>

## Notifications

### Get Maintenance Notifications

Retrieve maintenance notifications for the current user.

```http
GET /mantenimiento/api/notifications/
```

<ParamField query="limit" type="integer" default="20">
  Number of notifications to retrieve
</ParamField>

<ParamField query="unread_only" type="boolean" default="false">
  Only return unread notifications
</ParamField>

#### Response

<ResponseField name="notifications" type="array">
  Array of notification objects
  <Expandable title="Notification Object">
    <ResponseField name="id" type="integer">
      Notification ID
    </ResponseField>
    <ResponseField name="tipo" type="string">
      Notification type
    </ResponseField>
    <ResponseField name="mensaje" type="string">
      Notification message
    </ResponseField>
    <ResponseField name="leido" type="boolean">
      Read status
    </ResponseField>
    <ResponseField name="fecha" type="string">
      Notification date/time
    </ResponseField>
    <ResponseField name="relacionado_id" type="integer">
      Related object ID (e.g., work order ID)
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>

```javascript Request
fetch('/mantenimiento/api/notifications/?unread_only=true', {
  credentials: 'same-origin'
})
.then(res => res.json())
.then(data => console.log(data.notifications));
```

```json Response
{
  "notifications": [
    {
      "id": 789,
      "tipo": "OT_VENCIDA",
      "mensaje": "Work order OT-2024-1520 is overdue",
      "leido": false,
      "fecha": "2024-03-15T10:30:00Z",
      "relacionado_id": 1520

}
```

</CodeGroup>

## Individual Order Generation

### Generate Individual Work Order

Generate a single work order from a scheduled routine.

```http
POST /mantenimiento/proyeccion-generar/api/
```

<ParamField body="programacion_id" type="integer" required>
  Schedule/program ID
</ParamField>

<ParamField body="fecha" type="string" required>
  Date for the work order (YYYY-MM-DD)
</ParamField>

<ParamField body="activo_ids" type="array">
  Array of specific asset IDs to include
</ParamField>

#### Response

<ResponseField name="status" type="string">
  "success" or "error"
</ResponseField>

<ResponseField name="ot_id" type="integer">
  Generated work order ID
</ResponseField>

<ResponseField name="mensaje" type="string">
  Status message
</ResponseField>

<CodeGroup>

```javascript Request
fetch('/mantenimiento/proyeccion-generar/api/', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-CSRFToken': csrfToken
  },
  credentials: 'same-origin',
  body: JSON.stringify({
    programacion_id: 78,
    fecha: '2024-03-20',
    activo_ids: [123, 124]
  })
});
```

```json Response
{
  "status": "success",
  "ot_id": 1524,
  "mensaje": "Work order generated successfully"
}
```

</CodeGroup>

## Mobile Endpoints

The API includes mobile-optimized endpoints:

- `GET /mantenimiento/app/cronograma/` - Mobile maintenance schedule view
- `GET /mantenimiento/app/ot/{pk}/` - Mobile work order detail
- `POST /mantenimiento/app/ot/{pk}/iniciar/` - Start work order from mobile
- `POST /mantenimiento/app/ot/{pk}/finalizar/` - Complete work order from mobile
- `POST /mantenimiento/app/aviso/crear/` - Create maintenance notice from mobile
- `GET /mantenimiento/app/avisos/` - List user's maintenance notices
